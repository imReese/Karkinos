# Karkinos 收益与成本口径

本文记录 Karkinos Web 与后端 API 中收益、成本和基准价的基础口径。目标是让组合级数字、单只股票/基金数字、收益日历和持仓详情使用同一套可解释规则。

Karkinos 不提供投资建议，也不保证行情或券商成本永远完全一致；这些口径用于本地组合驾驶舱的可追溯计算与诊断。

## 核心分类

### 今日盈亏

今日盈亏回答的是：

> 今天市场价格变化，让当前持仓产生了多少浮动变化？

常规持仓使用：

```text
今日盈亏 = 当前持仓数量 * (当前行情价 - 当日基准价)
今日涨跌幅 = 当前行情价 / 当日基准价 - 1
```

这里通常不使用持仓成本价，因为成本价回答的是“买入以来赚亏多少”，不是“今天价格变动多少”。

当日基准价优先级：

1. `market_bars` 本地 OHLC 日线的上一交易日 `close`
2. `daily_close_snapshots` 中记录的上一交易日收盘快照
3. 实时行情上报的 `previous_close`
4. 更早的缓存行情兜底

如果本地 OHLC 日线与实时行情上报前收冲突，优先使用本地 OHLC 日线。这样收益日历、K 线和今日盈亏能使用同一个可复查的历史行情口径。

### 当日新买入持仓

当日新买入的股票或基金不能简单用昨日收盘价计算今日盈亏，否则会把买入前发生的价格变化算到当前账户上。

同一天买入且仍在持有的部分，应使用当日买入成本作为日内基准：

```text
当日新买入部分今日盈亏 = 当前持仓市值 - 当日买入总成本
```

当日买入总成本应包含成交金额和可确定费用，例如佣金、过户费等。券商实际成本价可能还包含不同舍入、最低佣金、过户费、规费或税费处理，Karkinos 应优先以已录入或导入的券商交割数据为准。

### 买入以来收益 / 浮动盈亏

买入以来收益回答的是：

> 从我的真实持仓成本到当前价格，这只资产当前浮动赚亏多少？

计算使用成本价：

```text
浮动盈亏 = 当前市值 - 持仓成本额
持仓成本额 = 当前持仓数量 * 成本价
收益率 = 浮动盈亏 / 持仓成本额
```

成本价应尽量贴近券商账户口径。对于 A 股，成本可能受以下项目影响：

- 佣金费率与最低佣金
- 过户费
- 规费或经手费等券商展示中合并处理的费用
- 印花税，通常卖出时产生
- 分批买入、卖出后的剩余持仓成本重算
- 四舍五入和券商内部精度

Karkinos 当前展示的成本价应作为本地账本成本视图；如果与券商不一致，应通过交割单或更完整费用规则修正账本，而不是用行情价格倒推。

### 已实现盈亏

已实现盈亏来自已经卖出、分红、赎回或其他确认事件。它不应该混入今日市场浮动，也不应该用来替代当前持仓的浮动盈亏。

```text
总收益 = 已实现盈亏 + 浮动盈亏
```

实际展示时，应明确标注“已实现”“浮动”“今日”三类，避免把不同时间维度混在一起。

### 现金

现金没有市场价格波动，因此不应产生浮动盈亏。

现金变化属于资金事件：

- 入金
- 出金
- 买入占用现金
- 卖出释放现金
- 分红到账
- 手续费扣减

现金可进入净值曲线和资产结构，但不应在 tooltip 或收益卡片中显示“现金浮动盈亏”。

## 组合级汇总

首页“今日盈亏”应按资产类别汇总：

```text
A股今日盈亏 = 所有股票今日盈亏之和
基金今日盈亏 = 所有基金今日盈亏之和
合计今日盈亏 = A股今日盈亏 + 基金今日盈亏 + 其他可定价资产今日盈亏
```

现金流入、现金流出、买入、卖出、分红和手续费是事件流，不应被伪装成市场涨跌。收益日历可以展示这些事件作为解释上下文，但市场盈亏和事件流入/流出必须分开。

## 数据可追溯原则

当同一只标的存在多种价格来源时，页面应展示或可诊断：

- 当前行情价
- 当前行情时间
- 行情来源
- 当日基准价
- 基准来源
- 是否使用缓存行情
- 缓存或缺价原因

如果缺少历史日线或基金净值，Karkinos 应标记缺价或缓存原因，而不是伪造收益。

## 示例

假设合成标的 SYNTH001 持仓 200 股：

```text
成本价 = 10.2500
当前价 = 10.7800
上一交易日 OHLC 收盘价 = 10.6000
```

今日盈亏：

```text
200 * (10.7800 - 10.6000) = +36.00
```

买入以来浮动盈亏：

```text
200 * (10.7800 - 10.2500) = +106.00
```

这两个数字可以不同，因为它们回答的是两个不同问题。
