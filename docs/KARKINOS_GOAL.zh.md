# Karkinos 目标

[English](KARKINOS_GOAL.md) | [路线图](ROADMAP.zh.md) | [架构](ARCHITECTURE.zh.md) | [中文文档](README.zh.md)

## 北极星

Karkinos 是面向中国市场的个人量化投研与交易平台，不是玩具回测器，也不是无人值守的盈利机器。

它应帮助一名严肃投资者减少情绪化错误，在使用前验证策略，优先控制下行风险，并让每个重要
决策都可审计。产品每天要回答的问题是：

> 根据我的组合、已持久化的行情与账户证据、风险限额以及已验证策略，今天应该买入、卖出、
> 持有、再平衡，还是不行动？为什么？

长期目标是建立一个人工监督、资本有界的个人投资操作系统。权限必须明确、会过期、可观察、
可逆，并且只能依据经过复核的证据扩展。

## 产品承诺

Karkinos 连接完整的投资操作闭环：

```text
研究想法
-> 可复现回测以及扣费后/OOS 证据
-> 人工复核的研究结论
-> 每日决策与目标组合
-> 数据、账户和风控门禁
-> paper/shadow 验证
-> 人工确认的受控执行
-> 券商与账本对账
-> 决策后复盘
-> 策略改进
```

每个转换都保留身份、来源、限制，以及操作为何被允许、阻断或要求复核的原因。

## 产品边界

### 财务事实

- 已持久化的 canonical facts 负责组合、账本、估值、绩效、账户、订单、成交与对账状态。
- 每个财务概念只有一个计算所有者；Web、报告与 AI 不得各自重新计算。
- 缺失、未发布、partial、stale、ambiguous 或 conflicting 的证据必须 fail closed，不得用看似
  合理的值替代。
- 券商证据不得静默修改生产账本。

### 研究与策略

- 回测绑定冻结数据集、参数、模拟成本、OOS 证据、限制与质量状态。
- 策略输出在通过数据、成本、风控、账户、paper/shadow 与操作员门禁前，只是研究证据。
- 策略代码只能提出信号或目标；不能导入或调用券商适配器，也不能自行授予权限。

### AI

- Provider、model、agent role、workflow、tool、evidence 与 memory 身份彼此独立。
- AI 工具默认拒绝，只能只读访问已持久化证据。
- 决策后 AI 复盘只能读取人工明确选择、绑定同一 valuation snapshot 与 ledger cutoff 的
  canonical 结果投影；不得重算收益，也不得把已复核观察转化为权限。
- 模型输出是带引用的非权威研究产物，不是账户事实、风控决策、资本授权、OMS transition、
  券商指令或权限。
- 在公式研究中，模型可以提出假设，但只有 allowlisted DSL 与 canonical backtest engine 可以
  计算结果。人工选择和最终处置不可省略；公式草稿不会注册生产策略或获得交易权限。
- 向外部导出证据需要单独、明确的人工决定，并排除凭证与权限状态。

### 执行

- 真实资金执行默认关闭。
- 逐单人工确认是默认的 live-like 模式。
- 未来任何有界 session 都必须明确、短期，并受账户、策略、标的、资本、换手、损失、回撤、
  速率、时间、数据、gateway 与对账门禁限制。
- Session 可以自动暂停或收窄，但不能自行续期、恢复、放宽或扩容。
- Kill switch、过期数据、账户不一致、connector 降级、未解决对账、policy 过期或限额耗尽都会
  阻断新提交。

## 成功标准

Karkinos 成功时应提供：

- 可复现、可解释的研究，而不是只展示醒目的回测收益；
- 包含不行动与需要复核结果的实用每日计划；
- 所有产品界面一致的组合与绩效事实；
- 可靠的 paper/shadow 与操作员 runbook；
- 每个 live-like 订单状态的明确证据与恢复路径；
- 可随时检查、暂停、收窄、过期或撤销的有界权限；
- 严肃个人投资者可以运行、理解和审计的本地开发与运营体验。

成功不等于保证盈利。工程目标是改善纪律、证据质量、风险控制、执行保真度以及扣除成本后的
学习能力。

## 非目标

- 投资建议或保证收益声明。
- 永久授权、无人值守的全账户交易。
- 策略或 AI 直连券商。
- 在 Karkinos 中保存券商密码。
- 自动扩大资本。
- 高频或低延迟交易。
- 将机构级多账户 OMS 或社区策略市场作为近期目标。
